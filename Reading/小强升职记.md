## 小强升职记

> 所谓的时间管理、所谓的高效工作、所谓的无压工作，并不是要让你在单位时间内完成比以前更多的事情，而是能让你把握住重点，保持平和的心态。

#### 效率

书中老付让小强记录每日时间开支，认识自己的时间黑洞。结果小强发现自己每天70%的时间都在做无意义的事情。

对于这点我深有体会。我从2017年的9月份开始开始用atimelogger记录自己时间的花费。我没有选择像柳比歇夫那样记录自己每一分钟的时间都花在了哪，只是记录了自认为"有效"的时间(编程、阅读、英语等)。结果呢，第一个月，我一共记录了69个小时，平均每天两个小时多一点。我在图书馆从早八点到晚九点坐一天记录的时间也不过三个多小时。效率太低了。

关于如何避免时间黑洞作者给出了以下几个小策略：

1. 彻底关闭聊天工具
2. 少看邮件
3. 给水杯里倒满水
4. 准备一张当天的任务清单，完成一项划掉一项
5. 解决最重要、最迫切的事情；
6. 按照番茄工作法，25分钟休息一下
7. 为任务设置最后期限。

### 目标

书中把目标放在了后面讲，但我认为开始就应该有一个清晰的目标，有了目标才能知道自己努力的方向。关于制定目标：

#### 0. 大卫·艾伦的六个高度

1. 原则(五万米)。首先思考自己的价值观和原则，这是个人的灵魂所在
2. 愿景(四万米)。包含3-5年的目标，我的目标是什么，谁达到了？他们如何达到的？
3. 目标(三万米)。比愿景更加细化，在一年内能完成，阶段性成果。
4. 责任范围(两万米)，为了将工作上和生活中的角色扮演好，需要执行的任务
5. 任务(一万米)，将注意力放在眼前的任务上
6. 下一步行动(跑道)。列入清单，逐一完成(下面详述)

#### 1. 正确描述目标

给自己定的目标要符合S.M.A.R.T标准。

- S：Specific，目标要具体
- M：Measurable，目标要是可测量的。这样才能了解自己的进度、更有动力前行。
- A：Attainable，可实现的，好高骛远，痴人说梦般的目标没有任何意义
- R：Realistic，可实行的，为了达到目标我该如何行动？
- T：Timebased，定好时间线，多长时间完成目标。

除了这五项以外，我们还要注意目标的时间限制。给自己定四五十年的目标其实没有任何意义，变数太大。要把目标定在目光可及的地方，完成后再制定下一个目标。2-5年的时间长度就是一个不错的选择。

#### 2. 分解目标

根据上面的方法写下来的目标可能像是这样的：我要在2年之内在BAT找到一份年薪15万以上的工作。

已经是比较清晰的目标了，那该如何去做才能达到目标？我们必须提前解决这个问题，才不至于在努力的过程中迷茫。

**目标的根基**

1. 把目标写下来！写下来有种神奇的功效，只是在头脑里想想可不行
2. 思考一下我为什么要达到这个目标？
   1. 为了提高生活质量
   2. 为了给自己的付出一个交代
   3. 为了让旁人刮目相看……
3. 我靠什么能达到这个目标？
   1. 我想要更好的生活，想给家人更好的生活，很想很想
   2. 我可以为此付出很大的努力
   3. ……
4. 确认自己完成目标的时间：两年！

**实现目标的具体步骤**

1. 写下要达到目标需要做的事情
   1. 掌握C/C++或Java
   2. 熟练运用Linux
   3. 算法、数据结构过关……
2. 将任务分配到每个月，列出每月计划表
3. 将每个月的任务细化到每一周

#### 3. 提高执行力

任务失败的原因大都是执行力不足，作者借老付分享了三个提升执行力的经历：

1. 找到最大的石头

   如果有一堆石头挡住了路，花一个小时把小石头都清理走车子还是无法过去。相反，如果我们花半个小时的时间找办法把最大的石头搬走，也许车子颠簸着也就过去了。如果不解决最重要的问题，小问题解决的再漂亮也没用。

2. 写下来

   把目标、要做的事情写下来。写下来有种神奇的功效，它可以给自己提醒、给自己立一个契约……把要做的事情写下来！

3. 拒绝第一次失败

   最终目标的失败往往是从第一次失败开始的。如果重视第一次，在之后的执行过程中可能会产生麻木的适应，对失败习以为常。

4. 重视失败

   当然，失败有时不可避免。但我们不能就这样听之任之。如果遇到了失败，要认真对待。分析失败的原因，重新审视计划是否有不合理的地方。

### 如何去做

有了目标就是有了一个清晰地方向。下一步就是如何把目标转化为行动，如何行动。

#### 四象限法则

所谓的四象限法则就是把事情分为四类：

- 第一象限：重要且紧急
- 第二象限：重要但不紧急
- 第三象限：不重要但紧急
- 第四象限：不重要且不紧急

**原则**

1. 尽量减少处理第三象限（不重要但紧急）的事情：运用"猴子法则"，把猴子甩给别人，尽量把事情交给别人去做
2. 第一象限（重要且紧急）的事情要尽量避免：其实第一象限的出现往往是因为第二象限没有处理好。比如说即将期末考试，复习就是一件重要且紧急的事情。但如果上课的时候有把“认真听课”这件重要但不紧急的事情做好，临阵磨枪的复习还有那么匆忙么？
3. 投资第二象限：第二象限的事情不急着去做但是对于个人提升帮助很大，例如学好英语、学习沟通技巧等等。这些事情什么时候都可以做，但正因为如此很多人都忽略他们而不去做。

**如何行动**

1. 根据自己的价值观评估事情的重要程度
2. 根据事情的截止日期评估事情的紧迫程度
3. 将所有事物放入四象限分析
4. 根据上述原则给不同象限的事情不同的处理办法

有了清晰的规划，才能有效的避免时间管理的几大杀手：拖延、犹豫不决、目标不明确。

#### 衣柜整理法

原则：每次只做一件事情。

将处理事情的过程分为五个步骤：收集、处理、组织、回顾、行动。

1. 收集，将所有需要处理的事情放入收集篮中，做好记录

2. 处理，将收集篮中的任务进行分类："可以执行"和"不能执行"。

   可以执行：

   1. 2分钟内可以完成，列入"待办事项清单"
   2. 需要多个步骤完成的，列入项目清单，将任务分解为可执行的行动，下面详述
   3. 可以交给别人完成的，把猴子甩给别人

   不能执行：

   1. 垃圾，千万不能去做的事情
   2. 将来可能去做的事情，比如约朋友吃饭，列入"将来某时清单"
   3. 参考资料，以后或许用得着的事情，将其归档

3. 组织，现在已经有了"将来某时清单"、"待办事项清单"和"项目清单"。接下来从待办项目清单中取出任务，根据类别分别执行：可以两分钟内完成的，立刻去做；可以推给别人的让别人去做……

4. 回顾

   每天开始都应该看一下今天要做的事情，做到心中有数。

   每周定期回顾：

   1. 回顾”将来某时清单“，如果有可以执行的去做
   2. 回顾项目清单，看看自己下一步要做什么，进度如何
   3. 回顾待办事项清单，掌握自己的完成情况
   4. 重新审视自己的目标，完成情况如何

5. 行动

**分解项目清单**

行动力差、效率低的重要原因就是我们把"项目"当做了"行动"。比如"学习Linux"就是一个项目，它让你无从下手，我们应该将其分解为可执行的行动，作为"下一步行动"。

**找到下一步行动的秘诀**

1. 动词开头，比如"打电话给某某"，"准备会议资料"
2. 内容清晰，像"准备会议资料"仍然不够清晰，要列出要准备什么资料、几点准备好等等
3. 描述结果，行动完成后应该能达到什么效果
4. 设定开始时间、周期和最后期限


流程图如下：

```flow
st=>start: 第二象限
op1=>operation: 项目
op2=>operation: 将行动写入清单
op3=>operation: 取出下一步行动
op4=>operation: 完成后从清单上划掉
cond=>condition: 是否还有下一步行动？
e=>end

st->op1->op2->op3->op4->cond(yes)->op3
```



**最重要的是，把上述的一切变成每一个工作日的习惯。**

### 处理突发事件

如果事件可以避免的：

1. 要事第一，要事第一！
2. 委婉而艰巨的拒绝
3. 争取下次机会

实在无法拒绝：

1. 立刻放松，别带着情绪去处理突发事件
2. 做好最坏的打算，调整心态
3. 默念口诀(世界如此美妙，我却如此暴躁，这样不好，不好)
4. 改变情绪，改变自己
5. 积极乐观(后两条有点虚……)
